# Governance

## Decision Making

Pharma Collective Platform uses a **lazy consensus** model for routine decisions:

1. A proposal is posted as a GitHub Discussion or PR
2. If no objections are raised within 7 days, it is considered accepted
3. For significant changes (data model changes, interface changes, severity changes in existing constraints), explicit approval from 2 core maintainers is required

## Core Maintainers

Core maintainers have write access to the repository and are responsible for:

- Reviewing and merging PRs
- Ensuring code quality and documentation standards
- Resolving disputes
- Releasing new versions

Becoming a core maintainer requires: sustained contribution over at least 3 months, demonstrated understanding of the architecture, and approval by a majority of existing maintainers.

## Domain Expert Reviewers

Domain experts review constraints and industry packs in their area of expertise. They do not need write access to the repository.

| Domain | Role |
|---|---|
| Pharma GMP | `@domain-pharma` team |
| Cell & Gene Therapy | `@domain-cgt` team |
| Food & Beverage | open — volunteers welcome |
| Semiconductor | open — volunteers welcome |

To join a domain expert team, open a GitHub Discussion and describe your expertise.
