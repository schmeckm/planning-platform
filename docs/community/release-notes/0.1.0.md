Pharma Collective Platform **0.1.0** is here.

This is the first release where the pieces fit together as a platform: a **canonical scheduling model**, a **plugin constraint engine**, **industry packs** for regulated manufacturing, **ERP/MES adapters**, a **REST API**, a **scheduling board UI**, and the new **shopfloor transparency module**. If you have been waiting for an open, explainable alternative to proprietary APS kernels — this is the foundation.

The headline for this release is not a single feature. It is the **contract**: planning knowledge becomes reusable packages instead of one-off consulting code.

Enjoy the release!

## Canonical data model in `planning-core`

Every adapter, constraint, and UI surface shares one model:

- `Order`, `Operation`, `Resource`, `Calendar`
- `Material`, `Batch`, `InventoryLevel`
- `AuditEntry`, `SimulationRun`

That means an SAP order, a CSV import, and a MES event all land in the same shape before constraints run. Integrators write mappings once; the solver never learns which ERP you use.

## Constraint framework

`planning-constraints` ships the plugin contract every contributor extends:

| Piece | Purpose |
|---|---|
| `PlanningConstraint` | Evaluate a rule against a plan snapshot |
| `ConstraintEngine` | Orchestrate evaluation order and aggregation |
| `ConstraintRegistry` | Register and discover plugins at runtime |
| `ConstraintTestCase` | Reproducible fixtures for validation |

Every violation returns a **human-readable message** and optional **suggested action** — explainability is not an afterthought.

## Industry packs

Two starter packs demonstrate how regulated domains plug in:

### Pharma (`planning-pharma`)

- `BatchReleaseConstraint`
- `CleaningValidationConstraint`
- `HoldTimeConstraint`
- `RMSLConstraint`
- `CampaignSequencingConstraint`
- `DeviationCheckConstraint`

### Cell & Gene Therapy (`planning-cgt`)

- `ChainOfIdentityConstraint`
- `VeinToVeinConstraint`
- `CryogenicStorageConstraint`
- `ApheresisWindowConstraint`

Each constraint is versioned, testable, and documented with business purpose and severity rationale.

## Adapters

`planning-adapters` defines the `PlanningDataAdapter` interface plus first connectors:

- **CSV** — spreadsheet and flat-file imports
- **Excel** — workbook-based staging for pilots

More ERP and MES connectors are tracked on the [roadmap](/community/roadmap).

## Shopfloor transparency module

`@PCP/planning-shopfloor` adds operational visibility without touching production write paths:

- MQTT ingest with **shadow message store**
- Board aggregation for live line KPIs
- PCP API routes `/api/pcp/v1/shopfloor/*` with `HaeShopfloorProvider` bridge
- Cockpit embed: Shopfloor Addon Board + MQTT admin tab (parity with legacy HAE portal)

See the [Shopfloor module guide](/modules/shopfloor) for architecture and configuration.

## Backward-incompatible changes

This is the **first public release** — there is no migration from a prior PCP version.

If you embed PCP inside an existing APS portal, treat all APIs as **v0.1** and pin versions until `1.0.0`.
